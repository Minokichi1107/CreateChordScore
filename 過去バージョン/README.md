# ChordPlayer - ギター練習用コード譜ツール

曲を流しながらコード譜を表示するローカルツールです。
ファイルサイズ制限なし・カポ対応・ギターダイアグラム表示。

---

## セットアップ（2ステップ）

### STEP 1：ライブラリのインストール

コマンドプロンプトで以下を実行：

```
pip install librosa numpy
```

※ 仮想環境を使う場合は先に有効化してください

---

### STEP 2：音声ファイルを解析

```
python analyze.py あなたの曲.mp3
```

完了すると `chord_data.json` が同じフォルダに生成されます。

オプション：
```
python analyze.py 曲.mp3 --output 好きな名前.json
python analyze.py 曲.mp3 --hop 1.0    # 1秒ごとに検出（デフォルトは0.5秒）
```

---

### STEP 3：ブラウザで再生

`chord_player.html` をダブルクリックしてブラウザで開き：

1. 「JSONファイル」ボタン → chord_data.json を選択
2. 「音声ファイル」ボタン → 元の音声ファイルを選択
3. 「PLAY」ボタンで再生開始！

---

## カポの使い方

右パネルの「カポ設定」でフレット番号を選ぶと：
- コード譜がカポポジションでのフォーム表示に自動変換
- ギターダイアグラムも対応したフォームに切り替わり
- 「実音: C」のように実際の響きも表示

例：曲がCメジャーで、カポ2フレットなら
→ Bメジャーポジションのフォームで表示

---

## ファイル形式

analyze.py が対応している音声フォーマット：
- MP3 (.mp3)
- WAV (.wav)
- FLAC (.flac)
- M4A (.m4a)
- OGG (.ogg)
- AIFF (.aiff)

ファイルサイズの制限はありません。

---

## 解析の精度について

- ライブラリ: librosa（クロマ特徴量 + KKプロファイル）
- ChordMiniAppの深層学習モデルよりは精度は落ちます
- メジャー・マイナー・7th・m7・sus4 の認識に対応
- ピアノ・アコースティックギター曲での精度が高め
- 複雑なバンドサウンドでは誤認識が増える場合あり

精度を上げるには --hop を小さくする（0.25など）か、
曲の音量を上げて解析すると改善することがあります。

## トラブルシューティング

### 「librosaがインストールされていません」と出る
```
pip install librosa numpy
```

### ブラウザでJSONを読み込んでも何も表示されない
- JSONファイルが正しく生成されているか確認
- ブラウザのコンソール（F12）でエラーを確認

### 解析に時間がかかる
- 長い曲（5分以上）は2〜5分かかることがあります
- --hop 1.0 を指定すると速くなります（精度は下がります）
