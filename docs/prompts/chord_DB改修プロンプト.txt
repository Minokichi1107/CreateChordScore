■ プロジェクト
https://github.com/Minokichi1107/CreateChordScore

■ 現状の問題

1. コード名の全角・半角問題
   現在、コード名検索が完全一致のため、以下のような場合に chord_db に登録されていてもダイアグラムが表示されません。

例
Ｃ＃ｍ7 → chord_db に存在しない
C#m7 → chord_db に存在する

つまり

全角コード
Ｃ＃ｍ7
Ｆ＃7

と

半角コード
C#m7
F#7

が別文字列として扱われています。

■ 要望①
コード検索前にコード名を正規化してください。

正規化内容
・全角英数字 → 半角英数字
・全角シャープ（＃） → #
・全角フラット（♭） → b
・前後の空白削除

例

Ｃ＃ｍ7 → C#m7
Ｆ＃7 → F#7

この正規化後の文字列で chord_db を検索するようにしてください。

■ 要望②
プロジェクト保存時も同じ正規化を行い、
コードは **必ず半角表記で保存**してください。

例

入力
Ｆ＃7

保存
F#7

■ chord_db の構造について

現在の chord_db は

{
"F#7": {...},
"C#m7": {...}
}

のような

「コード名 → 単一ダイアグラム」

の辞書構造になっています。

このため

F#7 のダイアグラムを複数登録すると

{
"F#7": diagram1,
"F#7": diagram2
}

となり、JavaScript仕様により後から登録したものだけが残ります。

■ 問題②

同じコードでも複数のダイアグラムを登録できるようにしたい。

例

F#7
・フォーム1
・フォーム2
・フォーム3

■ 要望③

chord_db を以下のような構造に変更してください。

{
"F#7": [
{ diagram: ... },
{ diagram: ... }
],
"C#m7": [
{ diagram: ... }
]
}

つまり

コード名 → ダイアグラム配列

に変更してください。

■ 影響範囲

・コード検索処理

・既存のコードダイアグラム表示を壊さない
・全角コード入力でも正常に動作するようにする
